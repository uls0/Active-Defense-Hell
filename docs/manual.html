01. INGENIERÍA DE PERSISTENCIA (LAYER 4)

HELL manipula directamente el stack TCP/IP para forzar al atacante a mantener la conexión abierta, agotando sus recursos de red y hilos de ejecución.

  • Zero-Window Tarpit: Congela el envío de datos mediante la manipulación del tamaño de ventana TCP.
  • TCP Keep-Alive Spoofing: Engaña al kernel del atacante enviando señales de mantenimiento para evitar el cierre de sockets.
  • Clamped Delivery: Fragmenta las respuestas en paquetes de 2 bytes para degradar la velocidad de procesamiento del bot.

02. AGOTAMIENTO DE CPU (LAYER 7)

Para atacantes basados en web o navegadores, HELL inyecta cargas de trabajo computacional extremo.

  • CPU Exhauster (WASM): Ejecución de bucles matemáticos infinitos mediante WebWorkers, saturando todos los núcleos del CPU del atacante.
  • Sticky Headers: Inyección de cientos de encabezados HTTP basura para desbordar los buffers de memoria del parser del atacante.
  • Infinite Redirect Loop: Laberinto de redirecciones dinámicas que atrapa a los crawlers automatizados.
