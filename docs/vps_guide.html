DigitalOcean Droplet Setup for HELL Project

1 Preparación del Droplet

Crea un Droplet con Ubuntu 22.04 LTS o superior. Se recomiendan al menos 2GB de RAM para manejar los hilos de saturación de HELL.

ssh root@tu_ip_servidor

2 Instalación de Dependencias

# Actualizar sistema

apt update && apt upgrade -y

# Instalar Docker & Docker Compose

curl -fsSL https://get.docker.com -o get-docker.sh

sh get-docker.sh

apt install docker-compose -y

3 Clonación y Configuración

git clone https://github.com/uls0/Active-Defense-Hell.git

cd Active-Defense-Hell

cp .env.example .env

# Configura tus API Keys y tu IP de Whitelist

nano .env

4 Lanzamiento (The Magic)

Ejecuta el despliegue en modo segundo plano:

docker-compose up -d --build

⚠️ Resolución de Problemas: Puerto 25 Ocupado

Si recibes un error indicando que el puerto 25 ya está en uso (Bind: address already in use), es probable que tu VPS tenga un servidor de correo (Postfix o Exim4) activo por defecto.

# 1. Identificar el servicio

lsof -i :25

# 2. Detener y deshabilitar el servicio (Ejemplo: Exim4 o Postfix)

systemctl stop exim4 postfix

systemctl disable exim4 postfix

# 3. Intentar el despliegue de HELL nuevamente

docker-compose up -d --build
